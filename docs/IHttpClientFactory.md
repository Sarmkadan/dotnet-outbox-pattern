# IHttpClientFactory

The `IHttpClientFactory` interface in the `dotnet-outbox-pattern` project defines a configuration-driven contract for creating and managing `HttpClient` instances with built-in resilience, proxy support, and request lifecycle management. It encapsulates HTTP client settings such as timeouts, retry policies, redirect behavior, and authentication credentials, providing both synchronous factory methods for client instantiation and asynchronous convenience methods for executing standard HTTP verbs directly without manual client disposal.

## API

### Configuration Properties

*   **`public TimeSpan Timeout`**
    Specifies the maximum duration allowed for an HTTP request to complete before cancellation. This value is applied to clients created by this factory.

*   **`public int MaxRetries`**
    Defines the maximum number of retry attempts for failed requests. This setting drives the resilience logic within `ResilientHttpClient`.

*   **`public string? UserAgent`**
    Gets or sets the default `User-Agent` header string to be included in all outgoing requests generated by this factory.

*   **`public Dictionary<string, string>? DefaultHeaders`**
    A collection of key-value pairs representing headers that are automatically added to every request issued by clients from this factory.

*   **`public bool FollowRedirects`**
    Determines whether the HTTP client should automatically follow redirection responses (3xx status codes).

*   **`public int? MaxRedirects`**
    Specifies the maximum number of redirects to follow if `FollowRedirects` is enabled. If null, a default limit may apply.

*   **`public ProxyConfig? ProxyConfig`**
    Contains detailed proxy configuration objects, used when complex proxy settings are required beyond a simple URL.

*   **`public string? ProxyUrl`**
    A string representation of the proxy server address. Used as a simplified alternative to `ProxyConfig`.

*   **`public string? Username`**
    The username credential used for basic authentication or proxy authentication.

*   **`public string? Password`**
    The password credential associated with the `Username` for authentication purposes.

*   **`public List<string>? BypassList`**
    A list of hostnames or IP addresses that should bypass the configured proxy settings.

### Factory Methods

*   **`public CustomHttpClientFactory`**
    Represents the concrete implementation type or configuration builder associated with this factory interface, allowing access to specific instantiation logic.

*   **`public HttpClient CreateClient`**
    Instantiates and returns a new `HttpClient` configured with the properties defined in this interface (timeout, headers, proxy, etc.).
    *   **Return Value**: A new instance of `HttpClient`.
    *   **Throws**: May throw configuration exceptions if proxy settings are invalid or credentials are malformed.

*   **`public HttpClient GetNamedClient`**
    Retrieves a named `HttpClient` instance, potentially leveraging internal pooling or specific named configurations distinct from the default `CreateClient` output.
    *   **Return Value**: An existing or new `HttpClient` instance associated with a specific name.
    *   **Throws**: Throws if the requested named client configuration does not exist.

*   **`public ResilientHttpClient`**
    Provides access to a specialized `HttpClient` wrapper that enforces the `MaxRetries` policy and handles transient failures automatically.

### Execution Methods

*   **`public async Task<HttpResponseMessage> GetAsync`**
    Sends an HTTP GET request to the specified URI and returns the response.
    *   **Parameters**: Implicitly uses the configured base address and headers; typically accepts a `string` or `Uri` target (signature inferred from standard usage).
    *   **Return Value**: A `Task<HttpResponseMessage>` representing the asynchronous operation.
    *   **Throws**: Throws `HttpRequestException` on network failure; may throw timeout exceptions if `Timeout` is exceeded.

*   **`public async Task<HttpResponseMessage> PostAsync`**
    Sends an HTTP POST request with content to the specified URI.
    *   **Parameters**: Accepts the target URI and `HttpContent` payload.
    *   **Return Value**: A `Task<HttpResponseMessage>` representing the asynchronous operation.
    *   **Throws**: Throws `HttpRequestException` on failure; respects `MaxRetries` for transient errors.

*   **`public async Task<HttpResponseMessage> PutAsync`**
    Sends an HTTP PUT request with content to the specified URI.
    *   **Parameters**: Accepts the target URI and `HttpContent` payload.
    *   **Return Value**: A `Task<HttpResponseMessage>` representing the asynchronous operation.
    *   **Throws**: Throws `HttpRequestException` on failure; respects `MaxRetries` for transient errors.

*   **`public async Task<HttpResponseMessage> DeleteAsync`**
    Sends an HTTP DELETE request to the specified URI.
    *   **Parameters**: Accepts the target URI.
    *   **Return Value**: A `Task<HttpResponseMessage>` representing the asynchronous operation.
    *   **Throws**: Throws `HttpRequestException` on failure; respects `MaxRetries` for transient errors.

## Usage

### Example 1: Configuring and Creating a Resilient Client
This example demonstrates configuring the factory with retry policies, proxy settings, and custom headers before generating a resilient client instance.

```csharp
var factory = new CustomHttpClientFactory
{
    Timeout = TimeSpan.FromSeconds(30),
    MaxRetries = 3,
    UserAgent = "OutboxPattern/1.0",
    ProxyUrl = "http://proxy.internal:8080",
    BypassList = new List<string> { "localhost", "127.0.0.1" },
    DefaultHeaders = new Dictionary<string, string>
    {
        { "X-Request-ID", Guid.NewGuid().ToString() }
    }
};

// Obtain a client that automatically handles retries based on MaxRetries
var resilientClient = factory.ResilientHttpClient;

try 
{
    var response = await resilientClient.GetAsync("https://api.example.com/data");
    response.EnsureSuccessStatusCode();
}
catch (HttpRequestException ex)
{
    // Handle final failure after retries exhausted
    Console.WriteLine($"Request failed: {ex.Message}");
}
```

### Example 2: Direct Execution with Authentication
This example utilizes the factory's direct execution methods to perform a POST request with basic authentication credentials configured at the factory level.

```csharp
var factory = new CustomHttpClientFactory
{
    Username = "service_user",
    Password = "secure_password_123",
    FollowRedirects = true,
    MaxRedirects = 5
};

var payload = new StringContent("{\"event\": \"order_created\"}", Encoding.UTF8, "application/json");

// Execute POST directly without manually managing the HttpClient lifecycle
var response = await factory.PostAsync("https://api.example.com/events", payload);

if (response.IsSuccessStatusCode)
{
    var content = await response.Content.ReadAsStringAsync();
    Console.WriteLine($"Event processed: {content}");
}
else
{
    Console.WriteLine($"Failed with status: {response.StatusCode}");
}
```

## Notes

*   **Thread Safety**: While `HttpClient` is generally thread-safe, the configuration properties (e.g., `Timeout`, `ProxyUrl`) on the factory instance itself should be considered immutable after initialization in a multi-threaded environment. Modifying these properties while `CreateClient` or execution methods are active may lead to inconsistent client behavior.
*   **Resource Disposal**: When using `CreateClient` or `GetNamedClient`, the caller is typically responsible for disposing of the returned `HttpClient` unless the implementation explicitly documents internal pooling management. However, when using the direct execution methods (`GetAsync`, `PostAsync`, etc.), the factory manages the internal client lifecycle and disposal automatically.
*   **Proxy Precedence**: If both `ProxyConfig` and `ProxyUrl` are set, the implementation logic (likely within `CustomHttpClientFactory`) will determine precedence. Typically, the strongly typed `ProxyConfig` object overrides the simple string `ProxyUrl`.
*   **Retry Logic**: The `MaxRetries` property only affects requests made through `ResilientHttpClient` or the direct execution methods (`GetAsync`, etc.) if they internally utilize the resilient wrapper. Standard clients created via `CreateClient` may not automatically enforce retries unless wrapped manually.
*   **Null Handling**: Properties such as `DefaultHeaders`, `BypassList`, and `ProxyConfig` are nullable. Implementations must handle null values gracefully, defaulting to no custom headers, no bypass list, or direct connection (no proxy) respectively.
